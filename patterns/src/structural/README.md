## Structural Patterns

### Objetivo
Os padrões estruturais têm como foco a composição e organização das classes e objetos, permitindo formar estruturas maiores e reutilizáveis.  
Eles ajudam a definir como os objetos se relacionam e se agrupam, mantendo o sistema flexível e fácil de entender.

### Benefícios
* Reduzem o acoplamento entre componentes;
* Permitem combinar objetos e classes de forma transparente;
* Facilitam a extensão do sistema sem alterar classes existentes;
* Promovem reutilização de código e clareza na arquitetura.

### Exemplos de padrões estruturais
* **Adapter**: Permite que interfaces incompatíveis trabalhem juntas;
* **Decorator**: Adiciona responsabilidades a objetos de forma dinâmica;
* **Facade**: Fornece uma interface simplificada para um conjunto de classes;
* **Composite**: Trata objetos individuais e composições de objetos de forma uniforme;
* **Proxy**: Controla o acesso a um objeto, podendo adicionar comportamento extra;
* **Bridge**: Separa abstração e implementação, permitindo variar ambas independentemente;
* **Flyweight**: Compartilha objetos para economizar memória.
